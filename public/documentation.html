World best todo lsit backends

Here are the routes we support

  • GET /todo

    This route will return a summary of all the todo items that we have stored

  • GET /todo/:id

    This route will return the details of a single todo item that matches the id in the path parameter. If no item is found, it will return null

  • POST /todo

    This route will add a todo item to the list. Include the description

    { "task":"Feed the squirrel", "description": "Make sure to use the organic, free range nuts we have." }
  • PUT /todo/:id

    This route will update single todo.

  • delete /todo/:id

    This route will delete todo
